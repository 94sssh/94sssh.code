## 이더넷과 무선 LAN

### MAC 주소 테이블 관리

하나의 포트에 하나의 MAC 주소만 등록되는 것은 아니다.  
여러 대의 스위치를 연결하면 하나의 포트에 복수의 MAC 주소가 등록된다.  
MAC 주소 테이블에 등록되는 MAC 주소 정보는 접속 포트가 바뀔 수도 있고, 대략 5분 정도의 제한 시간 값이 설정되어 있으므로, 영구적인 것이 아니다.

### 데이터 전송과 동시에 수신한다

**전이중 통신**: 데이터 송신과 수신을 동시에 하는 것  
**반이중 통신**: 송신과 수신을 동시에 할 수 없어, 서로 전환하면서 처리하는 것

레이어 2 스위치 기반의 이더넷 네트워크는 전이중 통신이 가능하다.  
버스형 토폴로지인 초기 이더넷은 반이중 통신이다. 초기 이더넷은 특정 시간에는 한 대만 데이터를 전송할 수 있고, 나머지는 수신만 가능하다.

전이중 통신을 실현하는 가장 단순한 방식은 데이터 수신용과 송신용으로 전송 매체를 나누어 사용하는 것이다.  
현재 레이어2 스위치를 이용한 이더넷이 데이터 수신용과 송신용을 나누어 전이중 통신을 가능하게 했다.  
레이어2 스위치와 PC의 이더넷 인터페이스는 UTP 케이블로 연결되어 있는데, 케이블은 4줄로 이루어져 있다. 총 4조 중 1조를 송신용, 1조를 수신용으로 나누어 사용한다.

### 케이블 없이 간편하게 네트워크를 만든다

무선 LAN 네트워크를 만들기 위해 `무선 LAN 액세스 포인트`와 `무선 LAN 인터페이스`가 필요하다.  
무선 LAN 인터페이스로 무선 LAN에 연결된 기기를 `무선 LAN 클라이언트`라고 표현한다.  
무선 LAN만으로는 통신이 완결되지 않는 경우가 보통이라 무선 LAN 액세스 포인트는 레이어2 스위치를 통해 유선 이더넷 네트워크와도 연결된다.

**인프라스트럭처 모드**: 무선 LAN 액세스 포인트를 경유해 데이터를 주고 받는 것. 무선 LAN은 무선 LAN 액세스 포인트를 경유해 데이터를 통신한다.

### 무선 LAN에도 규격이 많다

무선 LAN 규격의 차이는 이용하는 전파의 주파수 대역

|  규격 이름   |  책정시기   |      주파수대       |     전송속도      |
| :----------: | :---------: | :-----------------: | :---------------: |
| IEEE802.11b  | 1999년 10월 |      2.4GHz 대      |      11Mbps       |
| IEEE802.11a  | 1999년 10월 |       5GHz 대       |      54Mbps       |
| IEEE802.11g  | 2003년 6월  |      2.4GHz 대      |      54Mbps       |
| IEEE802.11n  | 2009년 9월  | 2.4GHz 대 / 5GHz 대 |    65-600Mbps     |
| IEEE802.11ac | 2014년 1월  |       5GHz 대       | 290Mbps ~ 6.9Gbps |

무선 LAN 규격인 IEEE802.11로, 상호 접속할 수 있음을 보증하는 의미로 쓰인 Wi-Fi가 현재는 무선 LAN 자체를 가리키는 표현이 되었다.

### 무선 LAN에 연결한다

**어소시에이션**: 무선 LAN에 연결하는 것, 어소시에이션에는 SSID가 필요  
**SSID**: 무선 LAN의 식별정보, 미리 무선 LAN 액세스 포인트에는 최대 32문자의 문자열로 SSID를 지정해 둔다. 암호화나 인증 등의 보안 설정은 SSID 별로 한다.

### 전파는 돌려쓴다

무선 LAN은 규격에서 정한 속도대로 통신할 수 있는 경우가 거의 없다.

**스루풋(실효 속도)**: 실질적인 통신 속도로, 무선 LAN의 스루풋은 규격상 전송속도의 절반 정도, 스루풋이 낮아지는 이유는 전파를 돌려쓰기 때문

어떤 순간에 무선 LAN에서 데이터를 전파에 실어 송신할 수 있는 것은 하나의 무선 LAN 클라이언트이다. 복수의 무선 LAN 클라이언트가 동시에 데이터를 보내면 전파가 중첩되어 데이터를 재구성할 수 없게 되는데, 이를 충돌이라 한다.  
무선 LAN에서는 CSMA/CA를 이용해 충돌을 방지하고, 전파를 돌려쓰도록 제어한다.

### 충돌이 일어나지 않게 데이터를 전송한다

**CSMA/CA**: 빠른 쪽이 이기는 방식으로 전파를 이용하는 메커니즘

1. 데이터를 송신하려고 할 때, 전파가 이용 중인지 확인
2. 전파가 미사용이면 다시 랜덤 시간 대기해 송신 타이밍이 겹치지 않게 해서 충돌 회피
3. 랜덤 시간 대기해도 전파가 미사용이면 데이터 송신, 데이터를 수신했다면 수신 확인 응답으로 ACK 반환

### 무선 LAN의 보안

무선 LAN 보안의 핵심은 인증과 암호화
